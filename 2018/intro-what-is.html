Введение в Haxe - что это такое
Введение в Haxe | 17.04.2018 |

В широком смысле Haxe (Haxe Toolkit) – это кросс-платформенный инструментарий с открытым исходным кодом, который содержит в себе следующие вещи:

  • Сам язык. Haxe – это высокоуровневый язык программирования с строгой типизацией, который транслируется «кросс- компилятором» в родной код платформы (Desktop, Web, Android, iOS, и т. д.) на другом языке программирования (C++, JS, PHP, Java, и прочие). Haxe легко выучить, если вы уже знакомы с объектно-ориентированными языками вроде тех же C++, Java, PHP, ActionScript3.
  • Транслятор кода, переводящий код с Haxe в код целевой платформы для последующей сборки или в байт-код виртуальных машин.
  • Стандартная библиотека, включающая работу с структурами и сериализацией данных, криптографией, файловой системой, базами данных и т. д. Также стандартная библиотека включает специфичное для платформ API, например, интерфейсы для работы с документом при трансляции в JavaScript.

Важно помнить, что Haxe – это не фреймворк, а инструментарий, который может быть использован для создания кросс-платформенных инструментов и фреймворков.

В чем же особенность языка, и какую пользу может принести разработка на нем? Haxe стремится быть универсальным языком, подходящим для разработки чего угодно и под что угодно, и, собственно, при помощи него можно быстро создавать код под множество систем. Сам по себе платформозависимый код, конечно, будет работать только для одной платформы, например, если вы пишете код для JavaScript, то собрать его же для C++ в том же виде не получится — есть обращение к интерфейсу JS, о котором C++ не имеет никакого представления. Но язык имеет богатые возможности для условной компиляции кода, с которой мы позже познакомимся поближе, то есть, можно написать часть кода, работающего в браузере, и часть работающего под десктопом, разграничить сборку, тогда мы получим универсальный код, работающий где угодно. Если мы посмотрим в любую библиотеку для Haxe, то увидим, что все интерфейсы, к которым в ходе работы обращается программист, дублируются для C++, JS и т. д., с использованием условной сборки, т. е., библиотека содержит множество бэкендов и один фронтенд для взаимодействия с человеком, пишущим код.

Отдаленно от особенностей сборки, Haxe – это мощный и выразительный язык, вобравший в себя многое как из классических объектно- ориентированных языков, вроде Java, так и из функциональных языков вроде OCaml (кстати, Haxe написан именно на Ocaml!), он имеет возможности для написания рефлексивного кода, обобщенного и метапрограммирования (продуманная система макросов, настройка трансляции кода). Поэтому в Haxe сожительствуют и работают вместе такие вещи, как классы и алгебраические типы данных.

Но все это не так интересно без наглядного примера. В качестве такого возьмем представленный на главной странице haxe.org:

class Test {
 static function main() {
  var people = [
   "Elizabeth" => "Programming",
   "Joel" => "Design"
  ];

  for (name in people.keys()) {
    var job = people[name];
    trace("$name does $job for a living!");
  }
 }
}

Как и во всех Java-подобных языках, конструкция вида class [имя] создает класс, в классе Test имеется статический метод main, т. е., при выборе Test в качестве базового, будет запускаться main. Несмотря на то, что Haxe – типизированный язык, он позволяет не указывать тип, в этом случае подразумевается, что используется особый тип Dynamic (данные в разный момент времени могут быть разного типа). Конструкция var [идентификатор], как в языках семейства ECMAScript (вроде JS, ActionScript), это объявление переменной, если надо создать переменную определенного типа, следует писать var [идентификатор] : [тип]. В примере объявляется people без явного указания типа, people является ассоциативным массивом, т. е., ключам соответствуют какие-то значения, это показано выражениями вида «строка» => «другая строка». Циклы по счетчику в Haxe являются не параметрическими, а циклами с итератором — в интерфейсе задается описание структуры данных и метод перемещения по ней, на основе этого интерфейса будет происходить цикл по конкретной структуре. Си-подобных for-циклов в Haxe нет вообще, и это поначалу вызывает затруднения и для некоторых может показаться серьезным недостатком, подобные вещи создаются как цикл в диапазоне: for ([id] in 0...[другое_число]) { … }, при этом создавать циклы от отрицательных чисел до положительных нельзя, но эта проблема легко решается при помощи while по счетчику и инкрементирования/декрементирования этого счетчика. Haxe позволяет производить вставку в строку из переменных по спецификации $: у нас имеется две локальные переменные name и job, и мы в праве указать $name и $job при выводе (trace – трассировка, вывод информации) для подстановки значений.
